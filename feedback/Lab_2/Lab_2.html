The Challenge¶

You have learned a lot of FSM design tools and skills and this challenge will require you implement everything you have learned to date to generate a more advanced control system for the zumo gate. This new control system will make use of the LED and Photodiode detector built into your gate base to detect objects.

Fixed Assumptions¶

  • The momentary button will be used as a manual gate activation signal (to take the place of the floor sensor). The gate will begin in a default position of "closed." One button press of BUTTON 1 will cause the gate to raise the gate and be open for 2 seconds, then lower automatically.
  • If an object is detected in the path of the gate by the photodiode as the gate is lowering, it will reverse and go to the fully up position. The gate will remain open for 2 seconds before lowering again, but only if the object blocking the gate is no longer present. Only objects that will be hit by the gate were it to reach the fully down position should cause the gate to rise.
  • If the object continues to be detected after 3 attempts to close, the gate will raise, the LCD will display "FAULT" and the nothing will happen until the system is manually reset.
  • To protect the motor, the motor will stop running when the gate arm is fully down or fully up.

Model Development: Object Detection¶

In order to detect an object in the gate arm's path, you will use a combination of an infrared (IR) LED and a photodiode. The IR LED is like the green LED on your workstation that illuminates when you apply power. In this case the light produced is not visible as it is in the Infrared (IR) spectrum. But how does and LED work? Similar to transistors, a light emitting diode uses semiconductors but is actually a bit simpler than a transistor.

void loop() {

  // BLOCK 1: Process Inputs
  //----------------------------------------
  int S_BTN1 = !digitalRead(BTN1); 
  int UP_PRESS = !digitalRead(UP_LATCH); 
  int DOWN_PRESS = !digitalRead(DOWN_LATCH); 
  int S_IR1 = analogRead(IR_1); 
  BTN1_PRESS = S_BTN1 && !BTN1_OLD; 

  if (S_IR1 > IR_thresh) { // checking if IR value is greater than defined threshhold
    detect = true;
  } else {
    detect = false;
  }

  //call timer
  TMR1 = Timer(waiting && pastUp, 2000);
  //call counter
  CNT1 = Counter(downToUp, 3, DOWN_PRESS);

  // BLOCK 2: State Transition Logic
  //----------------------------------------
  waitingToUp = waiting && BTN1_PRESS && !CNT1;
  upToWaiting = up && UP_PRESS || up && CNT1;
  waitingLatch = waiting && !BTN1_PRESS && !TMR1 || CNT1;
  waitingToDown = waiting && TMR1 && !CNT1;
  downToWaiting = down && DOWN_PRESS;
  upLatch = up && !UP_PRESS && !CNT1;
  downLatch = down && !DOWN_PRESS && !detect;
  downToUp = down && detect;


  // BLOCK 3: Update States
  //----------------------------------------
  waiting = waitingLatch || upToWaiting || downToWaiting;
  up = waitingToUp || upLatch || downToUp;
  down = waitingToDown || downLatch;

  if (upToWaiting == 1) { // if just transitioned from up to waiting...
    pastUp = 1;
  }
  if (downToWaiting == 1) { // if just transitioned from down to waiting...
    pastUp = 0;
  }


  // BLOCK 4: Set Outputs and Old Variables
  //----------------------------------------

  if (waiting == 1) { // waiting state...
    digitalWrite(37, LOW);
    digitalWrite(44, LOW);
    digitalWrite(46, LOW);
  }
  if (up == 1) { // lift gate up...
    digitalWrite(37, HIGH);

boolean CNT1_up = false;

bool Counter(bool CNT1_enable, int CNT1_preset, bool RST_enable) {

  // BLOCK 2: State Transition Logic
  //----------------------------------------
  CNTwaitToUp = CNT1_waiting && CNT1_enable && !CNT1; 
  CNTupToWait = CNT1_up && 1; 
  CNTwaitLatch = CNT1_waiting && !CNT1_enable;
